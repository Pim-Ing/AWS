Campus Lost & Found System

Simple • Cost-Effective • Scalable

Overview

This architecture demonstrates a scalable and reliable AWS Cloud deployment for a Campus Lost & Found system, using EC2, RDS, and S3 integrated for performance and availability.

Key Services

  • Amazon EC2 - Backend application server
  • Amazon RDS - MySQL/PostgreSQL database
  • Amazon S3 - Static files and photo storage
  • Amazon CloudFront - Global content delivery
  • Route 53 - DNS management
  • IAM - Access control and permissions

System Architecture

Essential AWS Services

This architecture uses only the core AWS services needed for a campus Lost & Found application, keeping costs low while maintaining security and scalability.

Core Services (Required)

  • Route 53: DNS management (~$0.50/month)
  • CloudFront: CDN for fast content delivery
  • S3 (x2): Static website + user-uploaded photos
  • VPC: Isolated network environment (free)
  • Internet Gateway: Public internet access (free)
  • EC2: Single backend server (~$15/month)
  • RDS: MySQL/PostgreSQL database (~$15/month)
  • IAM: Access control and permissions (free)

Estimated Monthly Cost: ~$35-50/month

Traffic Flow

1
Student accesses lostandfound.edu via Route 53 DNS
2
CloudFront CDN delivers cached content with HTTPS encryption
3
Static files (HTML/CSS/JS) served from S3 bucket
4-5
API requests routed through Internet Gateway to EC2 backend
6
EC2 queries RDS database for user data, posts, and messages
7
User photos uploaded to S3 via EC2 (using IAM role permissions)
8
Response sent back to user through same path

Security Features

  • Security Groups: Virtual firewalls controlling traffic
  • Private Subnet: RDS isolated from internet access
  • IAM Roles: Least-privilege access for EC2
  • HTTPS: Encrypted traffic via CloudFront
  • S3 Versioning: Photo backup and recovery
