Campus Lost & Found System

Simple • Cost-Effective • Scalable

System Architecture

Key Services

Area AWS Service Usage
Security IAM Defines roles and policies for secure access to AWS resources
Storage Amazon S3 Stores static website files and user-uploaded photos of lost/found items
Database Amazon RDS Stores user profiles, item posts, and messages in MySQL/PostgreSQL
Compute Amazon EC2 Hosts web application backend to handle login, posts, and messaging
Network/Delivery Amazon CloudFront Acts as CDN to deliver static files quickly with HTTPS encryption
DNS Route 53 Manages domain name resolution for lostandfound.edu

Security Groups (Orange Dashed Borders)

Web-Server-SG (Around EC2)

Inbound Rules: Port 80/443 from 0.0.0.0/0 (allows web traffic from internet)

Outbound Rules: Port 3306 to DB-SG, Port 443 to S3

Database-SG (Around RDS)

Inbound Rules: Port 3306 from Web-Server-SG ONLY

Outbound Rules: DENIED (database isolation - no outbound internet)

Purpose: Security Groups act as virtual firewalls controlling traffic between services with least-privilege access.

Arrow Legend

Solid Orange Arrow: User request flow and data queries
Dashed Blue Arrow: Permission grants and data uploads
Dashed Orange Arrow: Response path back to user

Monthly Cost Estimate

S3 (x2): ~$5-10/month

EC2: ~$15/month (t3.small)

RDS: ~$15/month (db.t3.micro)

Total Estimated Cost: ~$35-50/month

Application Flow

1 DNS Lookup

Student accesses lostandfound.edu via Route 53 DNS service

2 CDN Delivery

CloudFront CDN delivers cached content with HTTPS encryption for security

3 Static Content

Static files (HTML/CSS/JS) served from S3 bucket for fast loading

4-5 API Routing

API requests routed through Internet Gateway to EC2 backend server

6 Database Query

EC2 queries RDS database for user data, posts, and messages

7 Photo Upload

User photos uploaded to S3 via EC2 using IAM role permissions

8 Response

Application response sent back to user through the same path
